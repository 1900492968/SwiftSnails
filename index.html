Welcome to SwiftSnails.

SwiftSnails是一个分布式的参数服务器。 适用于一些参数结构比较固定的模型的分布式计算。 比如Logistic Regression，或者FM，神经网络这类模型，使用参数服务器都是很容易作分布式环境迁移的。

SwiftSnails中包含了参数PULL和PUSH最简单的接口，用户应用可以通过调用和封装这些简单接口来实现具体的逻辑。 项目还处于初期快速迭代的阶段，我们实现了Word2Vec的分布式版本作为Demo。

项目后续的发展方向是一些性能优化，和多一点的Demo。

欢迎有大神贡献代码，意见和建议。

任何一个项目都是为一个场景而服务，目前SwiftSnails实验的锅比较小，所以身量有限，如果应用的多的话，肯定会慢慢丰满起来。

项目介绍

目录

作为框架，SwiftSnail包含一些接口，当然代码量也比较少，即使全部看完，也比较容易。

一些有用的目录如下：

  • src : 项目源码
  • src/core : 项目核心代码
  • src/core/framework :框架模板（主要接口）
  • src/core/utils: 框架的一些底层代码
  • src/core/parameter: 有关参数的接口
  • src/core/transfer: 消息传递的框架，用户基本用不到
  • src/core/system: 参数服务器的内置逻辑，用户基本用不到
  • src/apps: 一些demo的代码，目前有word2vec
  • src/tools: 一些脚本工具，基本不能直接用，需要根据自己的环境修改
  • src/unitest: 项目早期的一些单元测试

接口

主要的接口包含在两个目录中：

  • src/core/framework
  • src/core/parameter

实际使用可以参看Word2Vec的Demo。

实验

这里主要分享分布式的Word2vec的一些实验数据，后续会有多一些的实验

  • 数据： 66.8G 词库规模：41w
  • window: 4
  • noise: 20
  • 维度：100
  • 集群节点数：8
  • 单机线程数：14
  • 单机配置：16核+16G内存
  • 运行中平均消耗：worker %1150 CPU 7.7G内存（可以修改本地参数缓存来优化） server %45 CPU 1.2G内存
  • minibatch: 10000句
  • AdaGrad, 初始学习率：0.5
  • 单轮迭代后词向量基本可用，单轮迭代3~4h
